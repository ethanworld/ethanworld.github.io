    结构体¶

    ngx_init_cycle:

    • 创建ngx_cycle_t

    • cycle->conf_ctx，申请内存，按ngx_max_module个指针大小

    • cycle->modules，调用ngx_cycle_modules从自动化ngx_modules拷贝所有模块

    • cycle->conf_ctx，是个数组，存放所有模块的配置结构体

    • ngx_conf_t的ctx字段，也指向这个配置数组

    ngx_conf_t是顶层配置，跟ngx_cycle_t是一个级别的。

    ngx_http_request_t:

    • ngx_http_alloc_request中创建

    • ctx字段方便全局获取任意模块的上下文，相当于是按request粒度，让各个模块记录一些状态，当然上下文结构体的申请也是使用r->pool，可以参考ngx_http_copy_filter

    • 通过ngx_http_set_ctx设置会话周期对应模块的上下文

    • 通过ngx_http_get_module_ctx获取对应模块的上下文

    ngx_http_module¶

    • 属于：NGX_CORE_MODULE
    • 主要是通过http命令触发ngx_http_block

    ngx_http_block

    • 创建：ngx_http_conf_ctx_t

    • 申请：main_conf数组内存

    • 申请：srv_conf数组内存

    • 申请：loc_conf数组内存

    • 各NGX_HTTP_MODULE模块创建三类配置

    • create_main_conf

    • create_srv_conf

    • create_loc_conf

    • 各NGX_HTTP_MODULE模块执行preconfiguration

    ngx_http_core_module

    • ngx_http_core_main_conf_t
    • ngx_http_core_srv_conf_t
    • ngx_http_core_loc_conf_t
